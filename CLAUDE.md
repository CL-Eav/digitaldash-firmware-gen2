## Project Overview

Embedded firmware for an automotive digital dashboard display running on an STM32U5G9ZJTXQ (ARM Cortex-M33, 160MHz). The dash renders LVGL-based UI to a 1024x600 display while reading vehicle telemetry via OBD-II and CAN bus. Target vehicle is a Ford Focus ST/RS 2013-2018.

## Build System

The primary build environment is **STM32CubeIDE** (Eclipse/CDT). Open the `.project` file in STM32CubeIDE to build and flash.

Submodule libraries and the UI layer use **CMake**:
```bash
# Build a library (example)
cd lib/lib_pid && cmake -B build && cmake --build build
```

Build metadata is auto-generated by Python scripts during the IDE build process:
- `scripts/gen_build_info.py` — produces `build_info.h` (version, commit, timestamp)
- `scripts/gen_boot_header.py` — produces bootloader header

Version is derived from Git tags (current: v1.0.10).

## Running Tests

Unit tests exist only for `lib/lib_pid` using the **Ceedling** framework:
```bash
cd lib/lib_pid
ceedling gcov:all utils:gcov   # Run tests with coverage
```
Coverage reports output to `build/artifacts/gcov/`. No other libraries have test suites.

## Architecture

### Firmware Layers (top to bottom)

1. **UI layer** (`ui/`) — LVGL screens and components. Key files: `ui.c`/`ui.h` (main UI), `alert.c`, `system_message.c`, `themes.c`, `themes/` (arc, digital, grumpy_cat, linear, radial, stock_strs), `images/` (compiled PNG assets). Images stored in external flash (Macronix 512Mb at `0xa0000000`).

2. **Application logic** (`lib/`) — Eight Git submodules:
   - `lib_digital_dash` — core dashboard orchestration
   - `lib_digitaldash_config` — USB-based configuration interface
   - `lib_vehicle_data` — vehicle data structures shared across libs
   - `lib_obdii` — OBD-II diagnostic protocol (CAN IDs: TX `0x7E0`, RX `0x7E8`)
   - `lib_can_bus_sniffer` — passive CAN frame monitoring
   - `lib_ke_protocol` — proprietary KE protocol (UART to ESP32 coprocessor)
   - `lib_unit_conversion` — unit conversion utilities
   - `lib_pid` — PID controller

3. **Display driver** (`Core/Src/lvgl_port_display.c`) — bridges LVGL to STM32 LTDC + DMA2D hardware.

4. **HAL/Drivers** (`Core/Src/*.c`, `Drivers/`) — STM32CubeIDE-generated HAL peripherals.

### Key Hardware Interfaces

| Peripheral | Purpose |
|---|---|
| FDCAN1 | CAN bus at 500 kbps, up to 28 filters |
| UART1 | ESP32 coprocessor (KE protocol) |
| I2C1 | ESP32 bridge |
| I2C2 | Serial EEPROM (24CW) |
| HSPI | External flash (Macronix MX25LM512) |
| LTDC + DMA2D | Display controller with GPU acceleration |
| TIM15 | Backlight PWM (4–100%) |

### Memory Map (from linker script)

- `0x08000000` — Bootloader
- `0x08012000` — Application FLASH (4096KB usable)
- `0x20000000` — RAM (3008KB)
- `0xa0000000` — External FLASH (65536KB, UI assets)

## Configuration

All firmware feature flags live in **`ke_conf.h`** (project root). Key settings:

- `VEHICLE` — set to `FORD_FOCUS_STRS_2013_2018` or `VEHICLE_GENERIC`
- `SPOOF_DATA` — use simulated vehicle data instead of live CAN
- `UI_ENABLED` / `DIGITALDASH_GRAPHICS` — enable display rendering
- `DIGITALDASH_DATA_ACQ` — enable vehicle data acquisition
- `UI_CONTAINER_DEBUG` — draw outlines around LVGL elements for layout debugging
- `UI_HOR_RES` / `UI_VER_RES` — logical UI canvas (1024×200); `ACT_HOR_RES` / `ACT_VER_RES` — physical display (1024×600)
- `DD_MAX_PIDS` — maximum simultaneous PIDs monitored (25)
- `BOOTLOADER_ACTIVATION` — enables DFU boot cookie mechanism

LVGL is configured in **`lv_conf.h`**: 32-bit color depth (XRGB8888), 256KB heap, 130 DPI.

## UI Regeneration

Image assets (PNG) are in `ui/images/` compiled as C arrays for external flash. Themes in `ui/themes/` define the visual style variants for the dashboard.
